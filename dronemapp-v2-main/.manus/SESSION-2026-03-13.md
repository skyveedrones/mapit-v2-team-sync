## One Remaining Action (Live Database)

The `users.role` enum in the live TiDB database still needs this DDL applied before the `client` role will persist for portal users:

```sql
ALTER TABLE users
  MODIFY COLUMN role
  ENUM('user','admin','webmaster','client')
  NOT NULL DEFAULT 'user';
```

Run this in your local terminal against the live DB when ready.

## Key Discoveries

- The Manus Platform deploys from its own **internal S3-based git**, not from GitHub. GitHub is a mirror/backup only. Commits pushed to `skyveedrones/dronemapp-v2` do not reach the live server automatically тАФ the **Publish** button in the Manus dashboard triggers a deploy from the platform's internal snapshot.
- The sandbox IP (`3.85.27.151`) is **not whitelisted** in TiDB Cloud's IP allowlist, so direct DB connections from the sandbox are blocked.
- The live database had **5 duplicate user records** (including `clay@skyveedrones.com`) that were cleaned up manually before the migration.
- The cached `.manus/db` query logs were stale and did not reflect the true state of the live database.
